## Design decisions
- **Shell mounts once**: the brief said the route wraps in `<AppShell>`, but
  `projects.$projectId` nests under `/projects` in TanStack file routing; wrapping again would
  double the chrome. Followed the established `knowledge`/`knowledge.$docId` idiom instead:
  parent renders AppShell + Outlet, child renders the bare screen.
- **FlowPanel indirection**: `ProjectScreen` renders `GlassPanel("THE FLOW", live)` itself and
  mounts `<FlowPanel branches/>` inside; `FlowPanel` holds the `./flow/BranchFlow` import.
  Screen tests `vi.mock("./FlowPanel")`, so they pass whether or not area C has landed and
  never touch BranchFlow internals.
- **One `nowS` per payload** for relative labels; only Pipelines ticks (1s) and only while a
  run is running.
- **No new colors**: everything from `tokens.ts` + the established status colors
  (`#ADD095` passed, `#FF8855` failed, commit-type chips mapped to existing tokens).
- 404 is NOT redirected (unlike unknown KB docs): SPEC §18 gives it a dedicated
  "This project isn't in the catalog" state with a back link; other errors get
  "Could not reach the Keel API" + retry (attempt counter re-arms `useAsync`).
- Loading = pulsing skeleton bars inside glass panels (reuses the `pulseDot` keyframe), no spinner.

## Route tree regen
The tanstackStart vite plugin regenerates `routeTree.gen.ts` on any dev/test boot (vitest loads
`vite.config.ts`). No manual `tsr generate` was needed — the tree picked up
`/projects/$projectId` automatically after the route file was created.
